## 二、逐文件深度分析

### 1. `mcp_servers/paper_search.py`（⭐⭐⭐⭐⭐ 最重要）

#### 作用
论文搜索是整个 Agent 的**第一入口**。用户说"帮我搜索 ISAC 论文"时，Agent 调用此工具从 Semantic Scholar / arXiv 获取**真实论文数据**，是防幻觉的关键一环。

#### 实现详解

**（1）双源搜索架构**
```
用户查询 → search_papers()  → Semantic Scholar API（引用数据全、结构化强）
         → search_arxiv()   → arXiv Atom API（最新预印本、开放获取）
```
两个数据源互补：SS 覆盖正式发表论文+引用统计，arXiv 覆盖最新未发表研究。

**（2）主动限速器（Proactive Rate Limiter）—— 亮点设计**
```python
MIN_REQUEST_INTERVAL = 0.2 if SS_API_KEY else 1.5
_last_request_time: float = 0.0

async def _rate_limit_wait():
    global _last_request_time
    now = time.monotonic()
    elapsed = now - _last_request_time
    if elapsed < MIN_REQUEST_INTERVAL:
        await asyncio.sleep(MIN_REQUEST_INTERVAL - elapsed)
    _last_request_time = time.monotonic()
```
- **设计思想**：不等 429 报错再重试（被动），而是**在请求前主动等待**（用户无感知）
- **对比传统方案**：被动 Backoff → 请求 → 429 → 等 1s → 请求 → 429 → 等 2s（用户看到两次错误）；主动限速 → 请求 → 自动等 1.5s → 请求 → 成功
- **面试考点**：这是一个**"用户体验优先"的工程决策**——牺牲微秒级延迟换取零报错体验

**（3）Exponential Backoff + 重试**
```python
for attempt in range(MAX_RETRIES + 1):
    try:
        response = await client.get(url, ...)
        response.raise_for_status()
        return response.json()
    except httpx.HTTPStatusError as e:
        if e.response.status_code == 429 and attempt < MAX_RETRIES:
            wait_time = INITIAL_BACKOFF * (2 ** attempt)  # 1s → 2s → 4s
            await asyncio.sleep(wait_time)
            continue
        raise
```
**双重保护**：主动限速（第一层）+ 被动 Backoff（第二层，兜底）

**（4）API Key 安全处理——踩坑记录**
```python
_raw_ss_key = os.getenv("SEMANTIC_SCHOLAR_API_KEY")
try:
    _raw_ss_key.encode("ascii")  # HTTP header 必须是 ASCII
    if _raw_ss_key not in ("你的key", "your_key_here", ""):
        SS_API_KEY = _raw_ss_key
except UnicodeEncodeError:
    logger.warning("API Key 包含非 ASCII 字符...")
```
- `.env.example` 中的中文占位符 `"你的key"` 会被 `load_dotenv` 原样加载
- 作为 HTTP Header 发送时触发 `UnicodeEncodeError`
- **工程教训**：永远不要假设环境变量的内容是合法的

**（5）防幻觉机制：可验证引用**
```python
ext_ids = paper.get("externalIds") or {}
doi = ext_ids.get("DOI", "")
arxiv_id = ext_ids.get("ArXiv", "")
verify_links.append(f"[DOI](https://doi.org/{doi})")
```
每篇论文都附带 DOI / arXiv ID，用户可以点击验证——**这是 Agent 防幻觉的关键设计**。

1. **Semantic Scholar API 的限流策略是什么？你怎么处理的？**

### 4. `knowledge/schema.py`（⭐⭐⭐⭐）

#### 作用

#### 节点 ID 生成算法

```python
@property
def node_id(self) -> str:
    normalized = self._normalize_label(self.label)
    if self.node_type == NodeType.PAPER:
        short_hash = hashlib.md5(normalized.encode()).hexdigest()[:8]
        return f"{self.node_type.value}_{short_hash}"

```

- 普通概念：`concept_ofdm`、`method_hybrid_beamforming`

def _normalize_label(label: str) -> str:

### 5. `knowledge/graph_analyzer.py`（⭐⭐⭐⭐）

#### 作用
基于图结构分析实现三大能力：**重要性评估 + 盲区检测 + 学习路径推荐**。

#### PageRank 在学术知识图谱中的含义
- 原始 PageRank：网页被越多链接引用，越重要
- 学术图谱：被越多论文/方法**引用或使用**的概念，越是核心基础概念
- **所以 PageRank 高 = 应该优先学习**

#### 综合重要性评分公式
```
importance = 0.4 × PageRank + 0.3 × degree + 0.2 × in_degree + 0.1 × betweenness
```
- **PageRank 权重最高（0.4）**：因为学术知识图谱的核心是"被引用的重要性传递"
- **介数中心性（betweenness，0.1）**：度量节点是否是"知识桥梁"——处于多少条最短路径上

#### 知识盲区检测——三种类型

| 盲区类型 | 检测条件 | 含义 |
|:---|:---|:---|
| `foundation_gap` | PageRank 前 30% 且属性 < 2 条 | 核心概念但你没深入了解 |
| `isolated_concept` | 度 ≤ 1（非 paper/author） | 知识点没有和其他知识建立联系 |
| `single_source` | 只从 1 篇论文了解 | 理解可能有偏见，需要交叉验证 |

#### 学习路径排序公式

```
priority = importance_score × 0.4 + gap_severity × 0.4 + topo_boost × 0.2
```

**拓扑排序的处理**：知识图谱可能有环（A uses B, B uses A），用 **SCC（强连通分量）缩点**后再拓扑排序：
```python
condensed = nx.condensation(self._graph)  # SCC 缩点
topo_list = list(nx.topological_sort(condensed))  # 无环后拓扑排序
```
