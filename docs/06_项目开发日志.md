## 关键架构决策记录 (ADR)

### ADR-001: 从单Agent起步，渐进演进到多Agent

- **决策**: Phase 0-2 使用单 Agent（Claude Code + MCP Tools），Phase 3+ 按需引入 sub-agent
- **理由**: 
  - 单 Agent 开发速度快、调试简单
  - MCP 架构天然支持渐进式扩展
  - 可以在 MCP Tool 内部启动 sub-agent 实现隔离
- **权衡**: 
  - 放弃了多 Agent 架构的"上下文隔离"优势
  - 换来了更快的迭代速度和更低的 debug 成本

### ADR-002: Phase 顺序调整（学习规划先于代码执行）

- **决策**: Phase 3 = 学习路径规划，Phase 4 = 代码执行
- **理由**:
  - 学习规划直接承接 Phase 2 的知识图谱，技术连贯性强
  - 代码执行需要 Docker 沙箱，工程难度更高，独立性强
  - 学习规划的用户感知价值高，可以尽早"自己用起来"

### ADR-003: 轻量级知识图谱存储方案

- **决策**: MVP 阶段使用 NetworkX + JSON，不用 Neo4j
- **理由**:
  - 个人使用场景论文量 < 100 篇，NetworkX 完全够用
  - 无需额外部署数据库服务
  - Schema 和数据格式可以后期平滑迁移到 Neo4j

### ADR-004: 论文来源三层体系

- **决策**: 支持在线搜索（Semantic Scholar/arXiv）+ 本地 PDF + Zotero 导入
- **理由**:
  - 在线搜索保证论文真实性（API 返回，非 LLM 编造）
  - 本地 PDF 是硕博生最常用的论文管理方式
  - Zotero 是学术圈主流文献管理器，后期集成增加黏性
