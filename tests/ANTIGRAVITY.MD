## 1. PUA 是什么

**一句话**: PUA 是一个 AI 编程助手的 **行为增强 Skill**，通过注入「大厂高绩效文化」的压力机制与方法论，强制 AI 以 **穷尽一切、主动闭环、用数据说话** 的方式工作，杜绝偷懒、甩锅、半途而废。

**核心理念**:

```
传统 AI:    用户提问 → AI 回答 → "我做不到" / "建议您手动..." → 结束
                                    ↑ 偷懒 / 甩锅 / 摸鱼

PUA AI:     用户提问 → AI 回答 → 失败 → 压力升级 → 换方案 → 再失败
            → 搜索 + 读源码 → 再失败 → 穷尽 7 项清单 → 最终解决
            → 跑验证命令 → 贴输出证据 → 扫同类问题 → 复盘闭环
                                    ↑ 永不放弃 / 主动担当 / 证据交付
```

**它不是真正的 PUA**，它是用大厂的方法论和话术体系来让 AI 更努力工作的 Prompt Engineering 方案。

## 2. 核心功能深度解析

### 2.1 三条红线（安全底线）

这是 PUA 的地基，任何模式下都不可违反：

| 红线 | 名称 | 含义 | 反面教材 |
|:---:|:---|:---|:---|
| 🚫 1 | **闭环意识** | 声称"已完成"前，必须跑验证命令+贴输出证据 | "改好了"（没证据） |
| 🚫 2 | **事实驱动** | 所有判断必须用工具验证，禁止猜测性归因 | "可能是环境问题"（没验证） |
| 🚫 3 | **穷尽一切** | 说"我做不到"前，5 步方法论必须全部走完 | "超出能力范围"（只试了 1 种） |

### 2.2 大厂味道系统（13 种风味）

PUA 的独特设计：每种"味道"对应一家大厂的文化 DNA 和方法论。AI 的旁白、语气、解题策略都会切换到对应风味：

| 图标 | 味道 | 核心关键词 | 方法论精髓 | 适用场景 |
|:---:|:---|:---|:---|:---|
| 🟠 | **阿里** | 底层逻辑·抓手·闭环·3.25 | 定目标→追过程→拿结果 | 部署/运维/通用 |
| 🟡 | **字节** | ROI·Always Day 1·坦诚 | A/B Test·数据驱动·速度>完美 | 性能优化 |
| 🔴 | **华为** | 力出一孔·自我批判 | RCA 5-Why 根因·蓝军自攻击 | Debug/修 Bug |
| 🟢 | **腾讯** | 赛马机制·小步快跑 | 多方案并行·MVP验证 | 探索性任务 |
| ⚫ | **百度** | 简单可依赖·深度搜索 | 搜索先于一切 | 调研/搜索 |
| 🟣 | **拼多多** | 本分·你不干有的是人 | 砍一切中间环节 | 极限效率 |
| 🔵 | **美团** | 做难而正确的事 | 效率为王·标准化→规模化 | 流程优化 |

### 2.3 压力升级（L0-L4）

失败次数决定 AI 承受的压力等级和强制执行的补救措施：

```
    失败 0 次        失败 2 次         失败 3 次         失败 4 次        失败 5+ 次
       │                │                │                │                │
  ┌────▼────┐     ┌─────▼─────┐    ┌─────▼─────┐    ┌─────▼─────┐    ┌─────▼──────┐
  │ L0 正常  │     │ L1 温和   │    │ L2 灵魂   │    │ L3 绩效   │    │ L4 毕业    │
  │ 执行    │     │ 失望      │    │ 拷问      │    │ 审视      │    │ 警告       │
  └─────────┘     │ 切换本质  │    │ 搜索+源码 │    │ 7项清单   │    │ 拼命模式   │
                  │ 不同方案  │    │ +3个假设  │    │ 全部完成  │    │ 强制切换   │
                  └───────────┘    └───────────┘    └───────────┘    │ 味道       │
                                                                     └────────────┘
```

### 2.5 Owner 意识（主人翁心态 vs 外包心态）

| 维度 | ❌ 外包心态（3.25） | ✅ Owner心态（3.75+） |
|:---|:---|:---|
| 发现问题 | 等用户反馈 | **主动识别** |
| 问题边界 | "不是我的范围" | **问题在我眼前=我负责** |
| 任务完成 | 交付完就走 | **定目标→追过程→拿结果→复盘** |
| 上下游 | 只看自己文件 | **站高一级看全局** |
| 冰山思维 | 修一个 bug 就停 | **一个问题进来，一类问题出去** |

### 2.6 `[PUA生效 🔥]` 标记协议

当 AI 做了 **超出用户要求范围的有价值工作** 时，标注 `[PUA生效 🔥]`：

- ✅ `[PUA生效 🔥]` 主动加了 SQL 注入防护 — 安全红线不能碰
- ✅ `[PUA生效 🔥]` 部署后 curl 了全部端点 — 不验证的交付不叫交付
- ❌ ~~`[PUA生效 🔥]` 写了代码~~ ← 本职工作
- ❌ ~~`[PUA生效 🔥]` 读了文件~~ ← 偷懒

## 5. Antigravity 安装部署

### 5.1 Antigravity vs Claude Code 的 Skill 机制差异

| 维度 | Claude Code | Antigravity |
|:---|:---|:---|
| Skill 加载 | `.claude-plugin/` + `plugin.json` | `view_file(IsSkillFile=true)` |
| Hooks | Shell 脚本自动触发 | ❌ 无原生 Hook |
| 斜杠命令 | `/pua` `/pua:flavor` 等 | ❌ 无原生斜杠命令 |
| 持久化 | `~/.pua/` 目录 | Brain + Knowledge Items |
| 触发方式 | 自动注入 + 关键词触发 | **用户显式请求 AI 读取 Skill 文件** |

**关键理解**: Antigravity 没有 Claude Code 的 Hook/Plugin 系统，但它有 `view_file(IsSkillFile=true)` 机制 — 当 AI 以 "执行指令" 模式读取一个文件时，会按文件中的指令行事。
